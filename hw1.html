code reading a stretchy buffer, aka a dynamic array, is a useful data struture. it holds a pointer to an array (the internal array), and has a function to push back (append) new elements. as new elements are pushed back, the internal array automatically grows (stretches). a reasonable implementation might allocate an internal array of length 16 on the first push back, and then double the length of the internal array as needed. std::vector is one implementation of a stretchy buffer. here is another: http://nothings.org/stb_ds/ - the capacity is the length of the internal array (the total number of slots elements *could* fill) - the length (aka size) of the stretchy buffer is the number of elements that are currently stored in the internal array (the number of slots that are actually filled) - ‚úÖ the capacity of a stretchy buffer is always greater than or equal its length
